## Типичные ошибки, которые НЕ ПОВТОРЯТЬ

### 1. Anthropic API: streaming обязателен при большом max_tokens
- При `max_tokens > 8192` API требует стриминг
- Использовать `client.messages.stream()` + `finalMessage()`, НЕ `client.messages.create()`
- Всегда обрабатывать `stop_reason === 'max_tokens'` — JSON может быть обрезан

### 2. JSON от Claude может быть обрезан
- Длинные транскрипции генерируют огромные JSON-ответы
- Всегда оборачивать `JSON.parse` в try/catch
- При ошибке парсинга пробовать "починить" JSON (закрыть массив/объект)

### 3. Определение имён спикеров
- "Сергей Николаевич?" (вопрос) = обращение К собеседнику, это НЕ имя говорящего
- "Это Ульяна" (утверждение) = самопредставление, это имя говорящего
- Промт для Claude должен содержать явные правила с примерами

### 4. Регрессии при добавлении фич
- Перед коммитом проверять что старый функционал работает, не только новый
- Таймкоды в транскрипции — хрупкая фича, легко ломается при изменении pipeline

### 5. Не делать публичные репозитории
- Это было поймано пользователем и вызвало потерю доверия. НИКОГДА.

## Стек и архитектура
- **Next.js** (App Router) + TypeScript
- **Deepgram** — транскрипция аудио (НЕ Claude)
- **Claude API** (Sonnet) — очистка текста + AI-фидбек
- **SQLite** (better-sqlite3) — база данных
- **Railway** — деплой, автоматический при push в main
- Том `/data` на Railway для аудиофайлов и БД

### Страницы приложения
- `/partnerka` — Партнёрка (промт Иры)
- `/sales` — ОП / Отдел продаж (промт Насти)
- `/callcenter` — КЦ / Колл-центр (промт Насти КЦ)
- `/transcription` — Транскрибация (только Deepgram, без AI)
- `/employees` — Сотрудники (отчёты по менеджерам)

### Pipeline обработки звонка
1. Загрузка аудио → сохранение файла в `/data/audio/`
2. Deepgram → сырая транскрипция с таймкодами (start/end)
3. Claude `cleanTranscript()` → очистка текста, сохранение таймкодов по id
4. Claude `generateFeedback()` → фидбек с подсветками, восстановление таймкодов по id
5. Сохранение в SQLite

### UI-дизайн
- Белый фон (НЕ тёмный)
- Лаконичный стиль
- Ключевые выводы — скрыты по умолчанию (details/summary)
- Транскрипция: слитный текст с инлайн-подсветкой, имена спикеров
- Каждый фрагмент кликабелен для перемотки аудио
